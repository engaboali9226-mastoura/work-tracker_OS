# Phase 2 - Step 075

# Self-Generation Registry Integration Test

## Status

Completed

## Goal

Prove that a component generated by Forge is discoverable by the architecture parser and projectable into the runtime component registry model.

## Reason

A manual probe proved the full path works:

~~~text
Forge Generator
↓
components/<component>/component.yaml
↓
Architecture Parser
↓
Component Registry Projector
↓
Registry entry
~~~

However, using the full registry generator writes runtime/component-registry.json and changes generatedAt.

This test avoids that side effect by using the registry projector in memory only.

## Important Detail

Forge tests run from the apps/forge workspace.

The architecture parser expects to run from the repository root.

The test temporarily changes the current working directory to the workspace root, then restores the previous working directory after cleanup.

## Test Added

~~~text
apps/forge/tests/self-generation-registry-integration.spec.ts
~~~

## What The Test Verifies

- Forge generates a temporary component.
- Architecture parser discovers the generated component.
- Component registry projector includes the generated component.
- Registry entry contains manifest, specification, contracts, implementation, and tests paths.
- Temporary probe component is removed after the test.
- runtime/component-registry.json is not written by the test.

## No Side Effects

The test does not write runtime/component-registry.json.
