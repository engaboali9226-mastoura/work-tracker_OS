# Documentation Boundaries

Status: Approved

---

## Global Documentation

Location:

docs/

Purpose:

Contains documentation describing the entire platform.

Examples:

- Vision
- Principles
- Architecture
- ADRs
- Roadmap
- Platform Workflows
- Platform Decisions

---

## Component Documentation

Location:

components/<component>/docs/

Purpose:

Contains documentation describing one component only.

Examples:

- README
- Decisions
- Usage
- Component Notes
- Examples

---

## Rule

Platform documentation MUST NEVER be stored inside a component.

Component documentation MUST NEVER describe the whole platform.
