# Architecture Principles

1.
Documentation First.

2.
Specification Before Implementation.

3.
Every Component Is Independent.

4.
Every Component Owns Its Business Logic.

5.
Every Component Has One Responsibility.

6.
Every Component Has Input Ports.

7.
Every Component Has Output Ports.

8.
Every Component Has Contracts.

9.
Every Component Can Be Tested Alone.

10.
Every Component Can Be Replaced.

11.
No Direct Component Coupling.

12.
Repository Is The Source Of Truth.
